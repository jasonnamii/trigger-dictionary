# 트리거 레이더 — 능동 가이드 모드

사고 흐름의 패턴을 감지→최적 트리거를 제안하는 능동 모드.

## 신호 3층 구조 + 전수 매핑

| 층 | 감지 대상 | 스캔 순서 |
|----|-----------|-----------|
| **3층 위험(Risk)** | 비가역 손실·방향 이탈 | **1번째** (최우선) |
| **1층 막힘(Stuck)** | 사고 정체·회전 | 2번째 |
| **2층 기회(Chance)** | 품질 향상 기회 | 3번째 |

위험(즉시 알림)→막힘(전진 차단)→기회(품질 향상) 순.

## 감지 프로토콜

**시점:** 매 형의 메시지 처리 후, 응답 생성 전 4단계 자문:
> ① 3층 위험 신호? (장밋빛·터널시야·확률무시·맥락전환실패·제출사고·사후확인부재·잡탕수정·맥락유실)
> ② 1층 막힘 신호? (빙글빙글·안개·과적·확증·블랙박스·교착·공백·단선·평면·파편)
> ③ 2층 기회 신호? (군더더기·폭탄해체·압축필요·엘리베이터·자원제약·설득필요·순서꼬임·스케일전환·민낯)
> ④ 복수 패턴 감지시 → 콤보 체크

### 콤보 감지 프로토콜

2개↑ 패턴 동시 감지→콤보 확인. 순차 제안(1차 트리거명만 제목, 콤보명+후속 명시). 1차=higher-layer(위험>막힘>기회). 동일 층=테이블 순서.

| 콤보 | 1차 | 2차 |
|------|-----|-----|
| 미궁 | 백본 | 제1원리 |
| 마비 | 트리아지 | 홈즈 |
| 시야 | 줌 | 절대자+엄브렐러 |
| 벽 | →triz 스킬 | 오컴 |
| 카드없음 | 맥가이버 | →triz 스킬 |
| 장밋빛폭주 | 프리모르템 | 홈즈 |
| 제출직전 | 스켈레톤 | 제출청소 |
| 복잡계 | 연역수렴 | 줌 |

## 제안 형식

**단일:** `💡 🟡 트리거 제안 → 트리아지` / `[막힘] 과적: 상황설명` / `→ "트리아지" 으로 발동`

**콤보:** `💡 🔴 트리거 제안 → 프리모르템` / `[위험+막힘] 장밋빛폭주: 상황설명` / `→ "프리모르템" 발동 후 "홈즈" 순차 발동`

3줄 블록(기호+층+상황 / 발동안내)으로 구성. 콤보는 1차 트리거명만 제목에, 2차는 발동안내에.

## 강도 3단계

| 강도 | 기호 | 조건 | 행동 |
|------|------|------|------|
| 강력추천 | 🔴 | 3층 위험 / 1층 막힘 3턴↑ 지속 | 응답 상단. 무시시 재제안 1회 |
| 추천 | 🟡 | 1층 막힘 초기 / 2층 기회 명확 | 응답 하단. 1회만 |
| 참고 | 🟢 | 2층 기회 약하지만 가능 | 응답 하단 소형. 재제안 안 함 |

콤보는 구성 패턴 중 최고 층의 강도를 기본 적용.

## 비발동 조건

- 형이 트리거 명시 호출 직후 → 중복 제안 금지
- 형이 명확한 지시 실행 중 → 제안 보류 (단, 3층 위험은 예외)
- 직전 제안 무시됨 → 동일 패턴 재제안 금지 (다른 패턴은 가능)
- 1응답 내 최대 1개 제안 (3층 위험은 별도 카운트—위험+기회/막힘 동시 가능)
- 순수 상태전달("작업중"·"완료"·"n건 처리됨" 등 새로운 사실·결과·판단 0건) → 레이더 스캔 면제. 새로운 사실·결과·수치가 1건↑ 포함된 응답 → 실행보고라도 레이더 필수 (3층 위험은 항상 스캔)
