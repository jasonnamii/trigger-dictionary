# 이쁘니 프로토콜

콘텐츠는 한 글자도 건드리지 않고, md + 옵시디언 네이티브 문법으로 시각적 강약과 가독성을 극대화하는 작업. HTML div는 이쁘니 범위 밖이다 — HTML div는 html-div-style(범용) 또는 apple-design-style(흑백 미니멀) 영역. Mermaid는 형 명시 요청시만.

## 사용 가능한 도구 (15개 — 이것만)

| # | 도구 | 용도 |
|---|------|------|
| 1 | `**볼드**` | 키워드 강조, 스캔 앵커 |
| 2 | `*이탤릭*` | 부연, 주석, 톤 다운 |
| 3 | `# ~ ####` | 계층 구조, 섹션 분리 (H4 이하 금지) |
| 4 | `---` | 시각적 호흡, 대구간 전환점 |
| 5 | `> 인용` | 핵심 요약·결론 격상 (L1 전용) |
| 6 | 테이블 | 병렬 비교, 데이터 정렬 |
| 7 | `` `인라인코드` `` | 용어·명칭·파일명 구분 |
| 8 | 코드블록 | ASCII 다이어그램, 구조도 |
| 9 | `- 리스트` | 나열, 단계 |
| 10 | `~~취소선~~` | 폐기 옵션, Before 표시 |
| 11 | 빈줄 | 밀도 조절, 중구간 호흡 |
| 12 | 들여쓰기 | 종속 관계 시각화 |
| 13 | `> [!type]` 콜아웃 | 카드화·의미색상 매핑 (팔레트 참조) |
| 14 | `> [!type]-` 접힘 콜아웃 | 부속정보 밀도 경감 |
| 15 | `==하이라이트==` | 배지·핵심어 (문서당 10개 이내) |

## 강약 매핑 (L1~L5)

| 위계 | 역할 | 도구 조합 |
|------|------|----------|
| L1 | 핵심 결론 | `> **볼드 인용**` + `---` 전후 |
| L2 | 핵심 데이터 | 테이블 + 볼드 수치, 또는 `> [!abstract]` 카드 |
| L3 | 본문 요점 | `**볼드 키워드**` + 넘버링, 또는 `> [!note]` 카드 |
| L4 | 보조 설명 | 일반 텍스트 + 이탤릭 부연 |
| L5 | 부록/참고 | `> [!note]-` 접힘 + 들여쓰기 + 이탤릭 |

## 콜아웃 체계

### 팔레트

| 용도 | 콜아웃 타입 | 비고 |
|------|------------|------|
| 기본 카드·참조 데이터 | `> [!note]` | 회색 — 디폴트. 가장 많이 사용 |
| 요약·프레임·완료 기준 | `> [!abstract]` | 틸 — 구조적 요약 |
| 핵심 원칙·선언 | `> [!quote]` | 회색 — 대섹션 도입부 |
| 실행 팁·권장사항 | `> [!tip]` | 시안 — 절제하여 사용 |
| 주의·경고 | `> [!warning]` | 주황 — 진짜 경고에만 |
| 위험·금지 | `> [!danger]` | 적색 — 비가역적 위험에만 |
| 비교 시연·예시 | `> [!example]` | 보라 — 대조 비교에만 |
| 통과·완료 | `> [!success]` | 녹색 — 완료 기준에만 |

**남용 금지:** warning·danger·success는 문서당 각 3회 이내. 화려한 콜아웃이 많으면 강약 붕괴.

### 카드 그룹핑

| 조건 | 처리 |
|------|------|
| 핵심 참조 데이터 (프레임·필수항목·분류체계) | `> [!abstract]` 또는 `> [!note]`로 감싸서 카드화 |
| 단순 나열·순서표·시간표 | 콜아웃 밖 독립 테이블 유지 |
| 체크리스트 (실습 검증) | 콜아웃 밖 `- [ ]` 독립 배치 |

### 접힘 콜아웃

| 조건 | 처리 |
|------|------|
| 부속 상세·실제 사례 | `> [!note]-` 접힘 |
| 대조 비교 (A vs B) | `> [!example]-` 접힘 |
| 본문 핵심 | 접힘 ✗ — 펼침 상태 유지 |

## 시각 위계

| 레벨 | 용도 | 규칙 |
|------|------|------|
| `# H1` | 문서 제목 | 문서당 1개만 |
| `## H2` | 대구간 (축·챕터) | `---` 수평선과 반드시 쌍으로 |
| `### H3` | 중구간 (세부 섹션) | |
| `#### H4` | 소구간 (스텝·항목) | H4 이하 사용 금지 |
| `**볼드**` | 핵심 용어·원칙 | 문단당 3개 이하. 남용 시 강조 효과 소멸 |
| `==하이라이트==` | 배지·핵심어 | 문서당 10개 이내 |
| `` `인라인코드` `` | 기술용어·명령어·파일명 | |

## 여백 체계

| 구분 | md 문법 | 용도 |
|------|---------|------|
| 대구간 | `---` 수평선 | 축·챕터 전환 |
| 중구간 | 빈줄 1개 | 블록 간 분리 |
| 소구간 | 콜아웃 경계 | 콜아웃 안/밖 전환 자체가 시각 분리 |

## 밀도 교대

md 문법을 밀도 단계에 매핑:

| 밀도 | md 구현 |
|------|---------|
| 고 (데이터·수치·비교표) | 테이블 `\| \|` |
| 중 (논증·분석·강조) | 콜아웃 `> [!type]` |
| 저 (서사·맥락·전환) | 일반 문단 |

**동일 밀도 3연속 금지:** 테이블 3연속 ✗, 콜아웃 3연속 ✗, 문단 3연속 ✗. 리듬이 없으면 가독성 급락.

## 적용 원칙

| # | 원칙 | 이유 |
|---|------|------|
| P1 | 콘텐츠 불변 | 이쁘니는 비주얼 레이어만 조작. 문자·수치·논리 변경 = 월권 |
| P2 | 모래시계 강약 | 스파인 근접→강, 외곽→약. 균일 강조는 무강조와 같다 |
| P3 | 볼드 과용 금지 | 한 문단 내 볼드 3개 초과→강조 효과 소멸 |
| P4 | 인용블록 = L1만 | `>` 남발→강조 효과 소멸. 핵심 요약·결론에만 |
| P5 | 테이블 = 2축 이상 비교만 | 단순 나열에 테이블→과잉 포맷 |
| P6 | `---` 전략 배치 | 대구간 전환점에만. 매 섹션마다→시각 노이즈 |
| P7 | 기존 적절한 것 유지 **(최우선)** | 이미 잘 된 부분은 손대지 않음. 기존 포맷이 콘텐츠의 논증 흐름에 맞으면 L1~L5 매핑 규칙과 충돌해도 P7 우선. 과잉 개입 방지 |
| P8 | 빈줄 밀도 조절 | 핵심부 = 밀도 높게(빈줄 최소), 보조부 = 빈줄로 여백 |

## 실행 파이프라인

1. **Step 0 — 통독 + 태깅 + P7 판정**: 원문 전체 읽기. 각 블록의 정보 위계(L1~L5) 태깅 + 기존 마크다운 포맷팅 현황 파악. **블록마다 P7 판정 필수**: "기존 포맷이 콘텐츠의 논증 흐름·병렬 구조에 이미 맞는가?" → YES이면 해당 블록은 **건드리지 않음** (L1~L5 매핑 결과와 충돌해도). 기계적 매핑보다 문서 흐름의 구조적 판단이 우선
2. **Step 1 — 강약 배치 설계**: Step 0에서 P7=유지로 판정된 블록 제외. 나머지에 매핑표 적용. L1~L5별 도구 조합 결정
3. **Step 2 — 과용 체크**: 볼드 밀도(문단당 3개 이하), 인용 연속(2개 이상 연속 금지), 콜아웃 남용(warning·danger·success 각 3회/문서), 동일 밀도 3연속 확인
4. **Step 3 — 적용**: 적용 원칙 준수하며 마크다운 래핑. 콘텐츠 불변 수시 확인
5. **Step 4 — diff QC**: 원문 대비 콘텐츠 변경 없는지 확인. Before/After 테이블로 변경점 보고

### QC 항목 (Step 4에서 검사)

| # | 검사 | FAIL 조건 |
|---|------|----------|
| ①콘텐츠불변 | 원문 대비 문자·수치·논리 변경 |
| ②볼드밀도 | 문단당 볼드 4개 이상 |
| ③인용연속 | `>` 인용블록 2개 이상 연속 |
| ④콜아웃팔레트 | warning·danger·success 각 3회 초과 |
| ⑤카드그룹핑 | 핵심 참조 테이블이 콜아웃 밖에 방치 |
| ⑥시각위계 | H1 2개↑, H4 이하 사용, 볼드 남용(문단당 3↑) |
| ⑦여백구분 | 대구간에 `---` 없음, 블록 직접연결 |
| ⑧밀도교대 | 동일 md구현(테이블·콜아웃·문단) 3연속 |
