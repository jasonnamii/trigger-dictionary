## 실행 프로토콜 (필수 — 스킵 금지)

**0. 축 실행 범위:**
모든 13축을 전수 실행한다. 모든 축에서 grep 패턴 검사를 수행하고, 오탐/미탐을 판별하여 실건수를 산정한다.

1. 대상 파일 전문을 1회 통독
2. 축 1→13 순서대로, 축별로 위 grep 패턴 실행 (축7은 grep+수동 병행, 축12는 수동, 축13은 Python). doc-tool 인용 대상 축은 스킵
   - **Python 일괄 스캔 가능:** `python scripts/trigger_scanner.py clean 대상파일.md` → 13행 테이블 + 축별 상세 자동 생성. 이후 오탐/미탐 판별은 수동 수행
3. **오탐/미탐 3단계 처리** (grep 결과마다 즉시 적용):
   - **①문맥 판별(오탐 제외):** grep 매칭이 문서 주제·일반 용어인 경우 제외. 예: AI 관련 문서에서 "AI" 매칭 → 축4 오탐, "줌 미팅" → 축3 오탐. 판별 기준: "이 단어를 삭제하면 문서 의미가 손상되는가?" Yes → 오탐(제외)
   - **②변형 탐색(미탐 보완):** grep 패턴이 잡지 못하는 변형 표현 추가 탐색. 예: "Claude" 외에 "클로드", "인공지능 생성" 등. 축별로 1회 변형 패턴 추가 탐색 수행. **축3 특별 주의:** 트리거명 외에 내부 작업 파이프라인 서술("독립 리서치 5축 + 보강 6축", "5건 수정, 3건 보강", "Phase 2 대조판정" 등)이 가장 빈번한 미탐 유형이다. "이 문장이 내부 작업 과정을 아는 사람만 이해할 수 있는가?"를 판별 기준으로 추가 적용
   - **③오탐/미탐 보고:** 13행 테이블에 "발견 건수" 외에 "오탐 제외" 열 추가. 오탐 제외 사유를 1줄로 명시
4. 결과를 **13행 고정 테이블**에 기록 — 미발견 축도 반드시 "✅ 0건"으로 명시 (미스캔 축 vs 0건 축 구분 목적). doc-tool 인용 축은 "✅ doc-tool CLEAN 인용"으로 표기
5. 1건↑ 축 → Before/After 제안 테이블 (위치·Before·After 3열)
6. 형 승인 대기 → 승인 후 일괄 적용
7. 축12 포맷정합 확인 → 축13 숫자정합 Python 검증
