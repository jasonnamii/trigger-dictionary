# 트리거 사전 — 24개 사고도구 즉시실행 엔진

> **발동 법칙:** 트리거 단어가 보이면 = 무조건 이 스킬 호출.
> 판단 금지. 기억 재구성 금지. 바로 호출.
> 24단어 + 8콤보 중 하나라도 대화에 등장하면 예외 없이 발동.
> **발동 시 §2 유사 트리거 1줄을 응답 상단에 선제 노출한 뒤 실행**.

## 1. 트리거 24종 정의

복합순서: 관점→분석→구조→판단→실행→전환. 충돌시 분석>구조>판단. **모든 행: `①②③④ 단계 → 산출: XXX` 형식.**

| 계 | 트리거 | 단계 → 산출 |
|----|--------|------------|
| 분석 | 홈즈 | ①관찰 ②이상징후 ③복수가설 ④증거대조 ⑤소거 → 산출: 결론+추론경로 |
| 분석 | 오컴 | ①경쟁가설 병렬 ②설명력 비교 ③가정최소 선택 → 산출: 채택가설+기각사유 |
| 분석 | 제1원리 | ①유추·관행 차단 ②환원불가 자명명제까지 분해 ③재조립 → 산출: 원리기반 재구성안 |
| 분석 | 베이지안 | ①사전확률 명시 ②새증거 강도반영 ③사후확률 갱신 → 산출: 확률표+갱신근거 |
| 분석 | 엄브렐러 | ①요소수집 ②절대자확장(동류전수) ③MECE분류 ④상위개념명명 → 산출: 우산도(그룹+명칭) |
| 분석 | 아날로지 | ①구조적패턴 추출 ②타도메인 성공사례 매핑 ③적용가능성 검증 → 산출: 매핑표+적용안 |

| 구조 | 백본 | ①결론먼저 ②역방향 전개 ③필수논거만 → 산출: 결론-근거 역피라미드 |
| 구조 | 스켈레톤 | ①비본질 제거 ②뼈대(논리흐름)만 잔존 → 산출: 골격도 |
| 구조 | SHE | ①Simple(핵심만 노출) ②Hide(부연 숨김·접근경로 보존) ③Embody(필요시점에 경험으로) → 산출: 노출계층+숨김규칙 |

### 1-1. 콤보 트리거 8종

콤보명 등장 = 구성 트리거 전부 순차 실행 (관점→분석→구조→판단→실행 순서 적용).

| 콤보 | 구성 | 상황 |
|------|------|------|
| 미궁 | 백본→제1원리 | 방향 상실 |
| 마비 | 트리아지→홈즈 | 할 일 많아 시작 불가 |
| 시야 | 줌→절대자→엄브렐러 | 시야 협소 |
| 벽 | triz→오컴 | 트레이드오프 돌파 불가 |
| 카드없음 | 맥가이버→triz | 자원·수단 부재 |
| 장밋빛폭주 | 프리모르템→홈즈 | 낙관 과잉 |
| 제출직전 | 스켈레톤→제출청소 | 납품 직전 점검 |
| 복잡계 | 연역수렴→줌 | 변수 많아 수렴 불가 |

## 2. 유사 트리거 제안 (발동 직후 상단 노출)

**규칙:** 트리거 발동 확정 시, 본격 실행 **전**에 응답 상단 1줄 필수 노출:

```
🔗 유사/보완: {A} · {B} · {C}
```

- **개수:** 2~3개 상한 (과잉 금지)
- **위치:** 발동 확인 직후, 실제 프로토콜 실행 전 1줄
- **목적:** 다음 수단 힌트 제공·트리거 체인 학습
- **콤보 발동 시:** 구성 트리거 전체의 유사를 중복 제거 후 상위 3개만
- **예시:**
  ```
  [엄브렐러 발동 감지]
  🔗 유사/보완: 절대자 · 스켈레톤 · 줌
  [엄브렐러 프로토콜 실행 ···]
  ```

## Gotchas

- **호출 생략:** 트리거 단어 등장 → 기억으로 재구성 = FAIL. 반드시 Skill tool 호출 후 §1 테이블 참조.
- **유사 노출 생략:** 트리거 발동 시 §2 유사 트리거 1줄 상단 노출 필수. 누락=FAIL. 매핑에 없는 트리거 조합=FAIL(§2-1 테이블 준수).
- **콤보 미인식:** 콤보명(미궁·마비 등)도 트리거. 개별 구성요소 전부 순차 실행.
- **복합 순서 무시:** 2개+ 트리거 동시 등장 시 관점→분석→구조→판단→실행 순서 적용.
- **수정4 레벨 과소판정:** 애매하면 상위 레벨(escalate-up).
- **POST_VERIFY 생략:** L1이라도 grep 잔존검색 필수. 검증 없는 수정 = FAIL.
- **이쁘니·TRIZ 이관:** 이쁘니→design-skill, TRIZ/트리즈/모순기술→triz. 이 스킬에서 처리 금지.
- **제출청소 축3 미탐:** 내부 작업 프로세스 용어(Phase N, 대조판정, N건 수정) + 분석 구조 표기(N층, N대 패턴, L0~L4, LIGHT/DEEP, 허브스포크, 게이트/폴백). 판별: "내부 과정 아는 사람만 이해?" Yes→실건수. 단, "3층 건물" 등 일반용법 제외. 축13 숫자정합은 Python 검증 필수.
