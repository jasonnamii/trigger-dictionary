## 1. 트리거 25종 정의

24개 = 분석7·구조7·실행4·판단2·관점2·전환2. 모순기술→triz 스킬로 이관. 이쁘니→design-skill로 이관. 복합순서: 관점→분석→구조→판단→실행. 계열 내 순서는 문제에 따라 유동. 충돌시 분석>구조>판단

| 계 | 트리거 | 행동 |

| 분석 | 홈즈 | 관찰→이상징후→복수가설→증거대조→소거→결론 |

| 분석 | 베이지안 | 사전확률 명시→새증거 강도반영→사후확률 갱신 |

### 1-1. 콤보 트리거 8종

콤보명이 대화에 등장하면 포함된 개별 트리거를 복합순서(관점→분석→구조→판단→실행)에 따라 순차 실행.

| 콤보명 | 구성 트리거 | 상황 |
|--------|------------|------|
| 미궁 | 백본→제1원리 | 방향 상실, 핵심이 안 잡힐 때 |
| 마비 | 트리아지→홈즈 | 할 일이 많아 시작 못 할 때 |
| 시야 | 줌→절대자→엄브렐러 | 시야가 좁아졌을 때 |
| 벽 | →triz→오컴 | 트레이드오프가 넘을 수 없을 때 |
| 카드없음 | 맥가이버→→triz | 자원·수단이 없을 때 |
| 장밋빛폭주 | 프리모르템→홈즈 | 낙관에 빠졌을 때 |
| 제출직전 | 스켈레톤→제출청소 | 납품·제출 직전 최종 점검 |
| 복잡계 | 연역수렴→줌 | 변수가 많아 수렴이 안 될 때 |

## Gotchas

- **"직접 할 수 있다" 판단으로 스킬 호출 생략:** 가장 흔한 실패. 트리거명이 대화에 등장하면 Claude가 프로토콜을 기억에서 재구성하려 함. 기억 재구성은 항상 불완전 — 반드시 이 스킬을 호출해야 정확한 프로토콜이 로드됨. **예:** "맥가이버"라고 했는데 스킬 호출 없이 "제약 상황에서 창의적으로..."라고 답하면 FAIL.
- **시맨틱 매칭 누락:** 트리거명 없이 개념만 서술하는 경우("있는 자원으로 해결해봐"=맥가이버, "뭐부터 해야 할지"=트리아지, "실패하면 어떻게 되지"=프리모르템). P3 시맨틱 패턴이 `trigger_scanner.py`에 정의되어 있으므로, 매칭이 애매할 때 스캐너를 실행하여 확인.
- **복합 트리거 순서 무시:** 2개 이상 트리거가 동시 등장 시 §G.복합순서(관점→분석→구조→판단→실행) 적용 필수. 먼저 보이는 트리거부터 처리하는 것이 아니라 우선순위에 따라 정렬.
- **콤보 트리거 미인식:** "미궁", "마비" 등 콤보명은 개별 트리거의 조합. 콤보명이 등장하면 포함된 개별 트리거를 전부 순차 실행해야 함.
- **수정4 레벨 과소판정:** 에이전트가 L3를 L2로, L4를 L3으로 판정해 게이트를 우회하는 패턴. escalate-up default를 반드시 적용 — 애매하면 상위 레벨.
- **POST_VERIFY 생략:** "간단한 수정이니 검증 불필요"라고 판단하는 패턴. L1이라도 grep 잔존검색은 필수. 검증 없는 수정 = FAIL.
- **이쁘니를 이 스킬에서 처리:** 이쁘니는 design-skill로 이관됨. "이쁘니 해줘" 호출 시 design-skill 발동 필요. 이 스킬의 references/에는 protocol-pretty.md가 더 이상 없음.
- **제출청소 축3 미탐:** 축3(내부언어)의 최빈 미탐 2가지: ①내부 작업 프로세스 용어(Phase N, 대조판정, N건 수정, 독립 리서치 등) ②**분석 구조 표기**(N층, layer N, N대 패턴/원리/공식, 축별 조사/통찰, L0~L4 교차, LIGHT/DEEP, 허브스포크, 게이트/폴백/스크리닝 등). 판별 기준: "이 문장이 내부 작업 과정을 아는 사람만 이해할 수 있는가?" Yes→실건수 포함. 단, "3층 건물" 등 일반 용법은 오탐으로 제외. 축13(숫자정합)은 금액·환율 존재 시 Python 검증 필수(0건 판정 금지). Before/After는 반드시 위치·Before·After 3열 테이블로 출력.
